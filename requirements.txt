# Argentina workshop — runtime requirements.
#
# Install with:
#     pip install -r requirements.txt
#
# Most pins are conservative lower bounds. The one version sensitivity
# we know about is ipympl + matplotlib (see the ipympl note below).

# --- Core (every notebook + every .py needs these) ----------------------
numpy>=1.20
torch>=1.10
matplotlib>=3.4

# --- HTTP client for the MotionGen synthesis endpoint -------------------
# Used by 4. drawing_to_mechanism.ipynb to POST {knn, path, types}
# to the path-synthesis backend. Skip if you only run notebooks 1-3.
requests>=2.25

# --- Image clipboard helper (Windows only) ------------------------------
# 4. drawing_to_mechanism.ipynb's Step 5 saves the BEFORE/AFTER figure
# and (on Windows) puts a CF_DIB on the clipboard so you can paste it
# into MotionGen. The encoding step needs Pillow. matplotlib usually
# pulls Pillow in transitively, but listing it explicitly avoids
# surprises on minimal environments.
Pillow>=9.0

# --- Jupyter (only if you want to run the notebooks interactively) ------
# The .py files don't depend on it.
jupyter>=1.0  ;  python_version >= "3.7"

# --- Interactive widget backend for the freehand canvas ----------------
# Used by notebooks 1 and 4 (`%matplotlib widget`). Older ipympl
# does NOT register its backend in a way matplotlib 3.10+ accepts,
# so make sure you have a recent one. On Colab, after installing
# you MUST do `Runtime > Restart session` once before the canvas
# cell will actually render — the notebooks' Setup cell prompts you
# to do this and uses `raise SystemExit` to enforce it.
ipympl>=0.9.4  ;  python_version >= "3.7"

# --- Cross-platform text-clipboard fallback for motiongen_export.py ----
# motiongen_export.py first tries the OS-native tool (clip.exe on
# Windows, pbcopy on macOS, xclip / xsel on Linux); pyperclip is only
# used if none of those are available. Safe to omit; the script then
# prints the JSON for manual copy.
pyperclip>=1.8
